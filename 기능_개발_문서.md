# 소방관 시뮬레이터 기능 개발 문서

## 프로젝트 개요

Unity URP 기반 1인칭 소방관 시뮬레이터.  
제한 시간 내에 화재를 진압하고 점수를 획득하는 게임.

---

## 개발 환경

- **엔진**: Unity (URP - Universal Render Pipeline)
- **언어**: C#
- **입력 시스템**: Unity Input System

---

## 구현된 기능 목록

### 1. 플레이어 시스템

#### PlayerController (`Assets/Scripts/PlayerController.cs`)
- WASD 이동 (Unity Input System `Move` 액션)
- 좌측 Shift 키로 달리기 (이동속도 2배)
- Space 키로 점프 (쿨타임 0.25초)
- Rigidbody 기반 물리 이동
- Animator 연동 (Grounded, MoveSpeed, Jump 파라미터)
- 키 입력:
  - `F`: 상호작용
  - `E`: 인벤토리 토글
  - `1~4`: 퀵슬롯

#### PlayerHealth (`Assets/Scripts/PlayerHealth.cs`)
- 최대 HP 100, Slider UI 연동
- 피해를 받으면 HP 감소
- HP 0 도달 시 `GameManager.GameOver()` 호출

---

### 2. 화재 시스템

#### FireManager (`Assets/Scripts/FireManager.cs`)
- 씬 시작 시 지정된 스폰 포인트에 화재 오브젝트 생성

#### FireZone (`Assets/Scripts/FireZone.cs`)
- Trigger 충돌로 플레이어 범위 감지
- 초당 `damagePerSecond` 만큼 플레이어에 지속 피해

---

### 3. 물 발사 시스템

#### WaterShooter (`Assets/Scripts/WaterShooter.cs`)
- 마우스 클릭(Fire 액션) 시 물 프리팹 발사
- 연사 속도: `fireRate` (기본 0.1초)
- 물 오브젝트 수명: `waterLifetime` (기본 1.5초)
- Player Input 컴포넌트 Send Messages 방식으로 `OnFire()` 호출

---

### 4. 게임 매니저

#### GameManager (`Assets/Scripts/GameManager.cs`)
- 싱글톤 패턴 (`GameManager.Instance`)
- 제한 시간: 90초, 타이머 카운트다운
- UI: 남은 시간(Text), 점수(Text)
- 게임 종료 조건:
  - 시간 초과 → Game Over
  - 씬의 모든 FireZone 제거 → Game Clear
- `AddScore(int)` 로 외부에서 점수 추가 가능

---

### 5. 맵 생성 시스템 (에디터 툴)

#### WallBuilder (`Assets/Editor/WallBuilder.cs`)
- Unity 메뉴: `Firefighter/벽 생성`
- 좌표 배열로 정의된 벽을 Cube 프리미티브로 자동 생성
- 벽 구조: `localScale = (두께 1, 높이 5, 길이)`
- `Quaternion.LookRotation` 으로 벽 방향 자동 설정

#### DepartmentStoreBuilder (`Assets/Editor/DepartmentStoreBuilder.cs`)
- Unity 메뉴: `Firefighter/맵 생성`
- PNG 도면 이미지를 읽어 픽셀 색상 기반으로 맵 자동 생성
  - 어두운 픽셀(R,G,B < 80): 벽 (Cube)
  - 그 외 픽셀: 바닥 (Quad, X축 90° 회전)
- 바닥 구조: Quad, `localScale = (1, 1, 1)`, `rotation = Euler(90, 0, 0)`

#### CreateMapPrefabs (`Assets/Editor/CreateMapPrefabs.cs`)
- Unity 메뉴: `Firefighter/맵 Prefab 생성`
- WallPrefab, FloorPrefab 및 기본 머터리얼 자동 생성
- `Assets/Prefabs/` 폴더에 저장

---

### 6. 머터리얼 타일링 시스템

벽과 바닥이 크기가 제각각이어서 하나의 머터리얼을 넣으면 텍스처가 늘어나는 문제 해결.  
`MaterialPropertyBlock` 방식을 사용해 머터리얼 인스턴스를 생성하지 않고 오브젝트별로 다른 타일링을 적용.

#### WallTilling (`Assets/Scripts/WallTilling.cs`)
- 벽 게임오브젝트(Cube)에 부착
- Inspector에서 벽마다 다른 머터리얼 할당 가능
- 벽 `localScale.z`(길이) / `localScale.y`(높이) 기준으로 타일링 자동 계산
- `textureSize`: 텍스처 1장이 표현하는 실제 크기(미터), 기본값 1
- `OnValidate`로 에디터에서 플레이 없이 즉시 결과 확인 가능
- URP 셰이더 프로퍼티: `_BaseMap_ST`

```
// 사용법
벽 오브젝트 선택 → Add Component → WallTilling
Wall Material: [머터리얼 드래그]
Texture Size: 1  (1m당 텍스처 1장)
```

#### FloorTilling (`Assets/Scripts/FloorTilling.cs`)
- 바닥 게임오브젝트(Quad)에 부착
- Inspector에서 바닥마다 다른 머터리얼 할당 가능
- 바닥 `localScale.x`(가로) / `localScale.y`(세로) 기준으로 타일링 자동 계산
- `textureSize`: 텍스처 1장이 표현하는 실제 크기(미터), 기본값 1
- `OnValidate`로 에디터에서 플레이 없이 즉시 결과 확인 가능
- URP 셰이더 프로퍼티: `_BaseMap_ST`

```
// 사용법
바닥 오브젝트 선택 → Add Component → FloorTilling
Floor Material: [머터리얼 드래그]
Texture Size: 1  (1m당 텍스처 1장)
```

**타일링 축 요약**

| 스크립트 | 오브젝트 | 가로 타일링 | 세로 타일링 |
|---|---|---|---|
| WallTilling | Cube | localScale.Z (벽 길이) | localScale.Y (벽 높이) |
| FloorTilling | Quad | localScale.X (바닥 가로) | localScale.Y (바닥 세로) |

**MaterialPropertyBlock 사용 이유**  
`renderer.material` 을 직접 수정하면 오브젝트마다 머터리얼 복사본이 생성되어 Draw Call이 증가함.  
`MaterialPropertyBlock` 은 머터리얼 원본을 유지한 채 오브젝트별 프로퍼티만 덮어씌워 성능에 유리함.

## 파일 구조

```
Assets/
├── Editor/
│   ├── WallBuilder.cs          # 벽 자동 생성 에디터 툴
│   ├── DepartmentStoreBuilder.cs # PNG 도면 기반 맵 생성
│   ├── CreateMapPrefabs.cs     # 기본 Prefab/머터리얼 생성
│   └── SceneSetup.cs
├── Scripts/
│   ├── PlayerController.cs     # 플레이어 이동/점프/입력
│   ├── PlayerHealth.cs         # 플레이어 HP 관리
│   ├── FireManager.cs          # 화재 스폰 관리
│   ├── FireZone.cs             # 화재 피해 트리거
│   ├── WaterShooter.cs         # 물 발사
│   ├── GameManager.cs          # 게임 시간/점수/승패 관리
│   ├── WallTilling.cs          # 벽 머터리얼 타일링
│   └── FloorTilling.cs         # 바닥 머터리얼 타일링
└── Prefabs/
    ├── WallPrefab.prefab
    ├── WallMaterial.mat
    ├── FloorPrefab.prefab
    └── FloorMaterial.mat
```
